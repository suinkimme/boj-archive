# NEXT JUDGE

저는 오랫동안 acmicpc.net에서 수많은 문제를 풀어왔고, 그 과정에서 알고리즘을 배우고 성장할 수 있었습니다. 그 토대를 만들어주신 최백준 님과 BOJ 팀에 항상 감사함을 느껴왔습니다.

그럼에도 불구하고 BOJ의 서비스 정책을 충분히 확인하지 않은 채 문제 데이터를 수집하고 공개 저장소에 올린 것은 명백히 잘못된 행동이었습니다. "공익적 목적"이라는 이유로 스스로를 합리화했지만, 그것이 저작권자와 출제자 분들의 권리를 침해하는 행위였음을 인정합니다. 변명의 여지가 없습니다.

요청하신 즉시 저장소의 모든 문제 데이터를 삭제하고, git 히스토리에서도 완전히 제거했습니다. 서비스에서의 문제 노출도 즉시 중단했습니다.

앞으로 NEXT JUDGE는 독자적으로 제작한 문제만을 사용하는 방향으로 다시 시작하겠습니다. BOJ와 혼동될 수 있는 어떠한 표현도 사용하지 않겠으며, 올바른 방식으로 서비스를 만들어나가겠습니다. 조금만 기다려 주시면 더 나은 모습으로 돌아오겠습니다.

다시 한번 깊이 사과드립니다.

---

## 앞으로의 방향

NEXT JUDGE는 처음부터 다시 시작합니다.

### 무엇이 달라지나요?

**이전:** BOJ의 문제 데이터를 가져와 다른 UI로 보여주는 서비스였습니다. 문제의 소유권은 BOJ와 출제자에게 있었고, 저희는 그것을 무단으로 사용했습니다.

**앞으로:** 모든 문제를 직접 제작하거나 커뮤니티 기여를 통해 만들어갑니다. 저희가 만든 문제만 서비스합니다.

### 어떻게 운영되나요?

NEXT JUDGE의 문제는 이 레포지터리에서 공개적으로 관리됩니다. 누구나 GitHub PR을 통해 문제를 기여할 수 있고, CI가 자동으로 검증합니다. 머지되는 순간 서비스에 즉시 반영됩니다.

기여 방법은 [CONTRIBUTING.md](./CONTRIBUTING.md)를 참고해주세요.

### 채점 방식

모든 채점은 **브라우저 안에서** 이루어집니다. 코드가 서버로 전송되지 않으며, Pyodide(Python → WebAssembly)와 wasm-clang(C/C++ → WebAssembly)을 통해 로컬에서 실행됩니다.
