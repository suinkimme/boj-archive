# 문제 기여 가이드

새 문제를 추가하거나 기존 문제를 수정하는 방법을 설명합니다.

## 폴더 구조

`challenges/<slug>/` 형태로 폴더를 만듭니다.  
`slug`는 문제를 식별하는 영문 소문자 + 하이픈 형식입니다 (예: `binary-search`, `stack-sort`).

```
challenges/
  binary-search/
    problem.md         # 문제 전체 (frontmatter + 마크다운 본문)
    solution.py        # 검증용 레퍼런스 솔루션
    testcases/
      01.in
      01.out
      02.in
      02.out
```

## problem.md 포맷

파일 상단의 `---` 블록에 메타데이터를, 그 아래에 마크다운으로 문제 본문을 씁니다.

~~~markdown
---
title: "두 수의 합"
time_limit: "1s"
memory_limit: "256MB"
tags:
  - math
  - implementation
samples:
  - input: "1 2"
    output: "3"
  - input: "-5 5"
    output: "0"
---

정수 A와 B가 주어질 때, 두 수의 합을 구하시오.

### 입력

첫째 줄에 두 정수 A, B가 공백으로 구분되어 주어진다.

- $-1{,}000 \leq A, B \leq 1{,}000$

### 출력

A와 B의 합을 출력한다.
~~~

### 필수 frontmatter 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `title` | string | 문제 제목 (전체 고유해야 함) |
| `time_limit` | string | 시간 제한 (예: `"1s"`, `"2s"`) |
| `memory_limit` | string | 메모리 제한 (예: `"256MB"`) |
| `tags` | string[] | 알고리즘 태그 목록 (아래 허용 목록 참고) |
| `samples` | array | 예제 입출력 (각각 `input`, `output` 문자열) |

### 본문 섹션

본문은 마크다운으로 자유롭게 작성할 수 있습니다.  
`---` 바로 아래부터 `### 입력` 이전까지가 문제 설명, `### 입력`, `### 출력` 헤딩 아래 내용이 각 섹션으로 표시됩니다.

태그는 아래 목록에서만 사용할 수 있어요. 목록에 없는 태그를 사용하면 CI에서 실패합니다.

| 브루트포스 | 정렬 | 이분 탐색 | 분할 정복 |

## 본문 작성

본문은 마크다운으로 자유롭게 작성할 수 있습니다.  
`---` 바로 아래부터 `## 입력` 이전까지가 문제 설명, `## 입력`, `## 출력` 헤딩 아래 내용이 각 섹션으로 표시됩니다.

## 테스트케이스 — 두 가지 방식

`testcases/`(수동)와 `gen.py + brute.py`(생성기) 중 하나 이상 있어야 합니다. 둘 다 써도 됩니다.

### 방식 1 — 수동 테스트케이스

```
testcases/
  01.in
  01.out
```

- 파일명은 `01.in` / `01.out` 형태로 번호를 매깁니다.
- `.in`마다 대응하는 `.out`이 반드시 있어야 합니다.
- 경계값, 최솟값/최댓값, 엣지케이스를 포함해 주세요.

### 방식 2 — 생성기 + 브루트포스 (권장, 로직이 복잡한 문제)

```
gen.py      # 랜덤 입력 생성기
brute.py    # 정답이 확실한 브루트포스 풀이
```

CI가 `gen.py 1`, `gen.py 2`, ..., `gen.py 100`으로 입력 100개를 만들고,  
`solution.py`와 `brute.py`의 출력을 비교합니다. 다르면 실패.

**gen.py 작성 규칙:**
- `sys.argv[1]`로 시드(정수)를 받아 랜덤 입력을 stdout에 출력합니다.

```python
import sys, random
random.seed(int(sys.argv[1]))

n = random.randint(1, 100_000)
a = [random.randint(-10**9, 10**9) for _ in range(n)]
print(n)
print(*a)
```

**brute.py 작성 규칙:**
- stdin으로 입력받고 정답을 stdout에 출력합니다.
- 느려도 되지만 반드시 정확해야 합니다.

## 레퍼런스 솔루션

`solution.py`는 의도한 최적 풀이를 작성합니다.  
리뷰어가 "이 솔루션이 문제 설명대로 올바르게 동작하는가"를 판단하므로 정확하게 작성해 주세요.

> 제출 코드와 동일한 환경(Pyodide, CPython 3.11 → WebAssembly)에서 실행됩니다.  
> `subprocess`, `threading`, 네트워크, 파일 I/O는 사용할 수 없습니다.  
> 자세한 내용은 [Python 실행 환경](docs/python-runtime.md)을 참고하세요.

## 기여 방법

1. 이 레포를 포크합니다.
2. `challenges/<slug>/` 폴더에 파일을 추가합니다.
3. PR을 생성합니다.

## CI 검증 항목

PR을 열면 자동으로 다음을 검사합니다.

| 항목 | 설명 |
|------|------|
| YAML 스키마 | 필수 필드 존재 여부, 타입 검사 |
| 중복 제목 | 기존 문제와 title 중복 여부 |
| 솔루션 실행 | `solution.py`가 모든 테스트케이스를 통과하는지 |

CI가 통과하고 리뷰어 1명이 승인하면 머지됩니다.  
main 브랜치에 머지되는 순간 DB에 자동 반영되어 즉시 서비스에 공개됩니다.
